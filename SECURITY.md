# Security Policy

## Supported versions

The project is in pre-production and follows best-effort support on the latest
`main` branch.

## Reporting a vulnerability

Do not open public issues for security vulnerabilities.

Preferred flow:

1. Use GitHub private vulnerability reporting (Security tab) when available.
2. If not available, contact repository maintainers directly.

Include:

- Impact summary
- Reproduction steps
- Suggested mitigation (if known)
